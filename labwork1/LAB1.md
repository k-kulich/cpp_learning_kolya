# Лабораторная работа №1: Анализ логов

## Задача

Реализовать утилиту командной строки для анализа лог-файла сервера (формат **access.log**).  
Утилита должна поддерживать следующие функции:

1. Вывод запросов, завершившихся с кодом ошибки **5XX** (т.е. статус 500–599).
2. Поиск промежутка времени (окна) заданной длительности, в течение которого количество запросов было максимальным.
3. Вывод **N** самых частых запросов, завершившихся с ошибкой 5XX.

Дополнительно программа должна уметь принимать аргументы командной строки для управления поведением.

---

## Аргументы командной строки

Утилита запускается так:

```
AnalyzeLog [ОПЦИИ] logs_filename
```

| Короткий | Длинный | Значение по умолчанию | Описание |
|----------|---------|-----------------------|----------|
| `-o path` | `--output=path` | – | Путь к файлу, в который будут записаны запросы с ошибками. Если не указан – ошибки не сохраняются. |
| `-p` | `--print` | – | Выводить запросы с ошибками также в `stdout` (терминал). |
| `-s n` | `--stats=n` | `10` | Вывести `n` самых частых запросов с ошибкой 5XX (в порядке убывания частоты). |
| `-w t` | `--window=t` | `0` | Найти и вывести временной промежуток длительностью `t` секунд, в который было максимальное число запросов. При `t = 0` расчёт не производится. |
| `-f time` | `--from=time` | минимальное время в логе | Временная метка (Unix timestamp), с которой начинается анализ. |
| `-e time` | `--to=time` | максимальное время в логе | Временная метка (Unix timestamp), до которой включительно производится анализ. |

**Важно:** название файла лога передаётся последним аргументом без ключа.

Примеры запуска:

```bash
AnalyzeLog access.log --output=result.txt
AnalyzeLog access.log --output=result.txt --print --stats 5
AnalyzeLog --stats=2 --window=60 --from=805821284 --to=807117284 access.log
AnalyzeLog -s 2 access.log
AnalyzeLog -w 10 access.log
```

---

## Формат лог-файла

Каждая строка файла имеет вид:

```
<remote_addr> - - [<local_time>] "<request>" <status> <bytes_send>
```

- `remote_addr` – IP-адрес клиента.
- `local_time` – дата и время запроса в формате `[dd/MMM/yyyy:hh:mm:ss Z]` (например, `[03/Jul/2024:10:50:02 -0400]`).
- `request` – строка запроса в кавычках (например, `"GET /shuttle/countdown/ HTTP/1.0"`).
- `status` – числовой код ответа HTTP (например, 200, 404, 500).
- `bytes_send` – количество отправленных байт.

Пример строки:

```
198.112.92.15 - - [03/Jul/2024:10:50:02 -0400] "GET /shuttle/countdown/ HTTP/1.0" 200 3985
```

## Требования к реализации

1. **Память:** файл лога может быть очень большим (гигабайты). Программа не должна загружать его целиком в память. Обрабатывайте строки потоково (читайте построчно).

2. **Модульность:** разбейте программу на логические функции:
   - парсинг строки лога;
   - фильтрация по статусу и временному диапазону;
   - расчёт статистики и поиск окна;
   - обработка аргументов командной строки.

3. **Стиль кода:** придерживайся [Google C++ Style Guide](https://google.github.io/styleguide/cppguide.html).

4. **Обработка ошибок:** предусмотри ситуации:
   - файл не найден или не может быть открыт;
   - некорректные аргументы;
   - неверный формат строки в логе (желательно игнорировать такие строки, но выводить предупреждение).

5. **Стандарт:** используй **C++17**. Разрешается только стандартная библиотека (STL), внешние библиотеки запрещены.

---

## Что должно получиться на выходе?

- **При использовании `-o`** – создаётся файл с перечнем строк (запросов), у которых статус 5XX.
- **При использовании `-p`** – те же строки дублируются в терминал.
- **При использовании `-s n`** – в терминал выводится таблица (или просто список) из `n` самых частых URL-ов среди ошибок 5XX. Например:
  ```
  /some/bad/url  - 150 раз
  /another/error - 87 раз
  ...
  ```
- **При использовании `-w t`** – выводится временной интервал (две даты) и количество запросов в нём. Например:
  ```
  Window: [03/Jul/2024:10:50:00 -0400] - [03/Jul/2024:10:51:00 -0400], requests: 234
  ```

Если заданы одновременно `-s` и `-w`, выводятся оба результата (сначала статистика, потом окно).

## Подсказки

- Для парсинга даты из строки используй `std::get_time` или разбирай вручную (формат фиксированный).
- Для преобразования даты в Unix timestamp можно использовать `mktime` (из `<ctime>`) после заполнения структуры `struct tm`.
- Для работы с опциями командной строки можешь написать свой простой парсер (перебор `argv`, но, в принципе, библиотеку напишем чуть попозже вместе). Не подключай сторонние библиотеки.
- Для хранения частот запросов удобно использовать `std::map` или `std::unordered_map`.
- Поиск окна можно реализовать методом двух указателей (скользящее окно) после сортировки времени запросов.
